# Security Policy

## Supported Versions

We only provide security updates for the primary production branch.

| Version | Supported          |
| ------- | ------------------ |
| Main    | :white_check_mark: |
| Legacy  | :x:                |

## Reporting a Vulnerability

As a commercial application, we take security very seriously. If you discover a security vulnerability within this project, please report it privately to our security team.

**Please do not report security vulnerabilities via public GitHub issues.**

### How to Report

Please email reports to security@nihongo.com with the following information:

1.  **Type of issue** (e.g., SQL injection, XSS, RLS bypass).
2.  **Steps to reproduce** the issue.
3.  **Potential impact** of the vulnerability.
4.  (Optional) Suggested fix or mitigation.

### Our Response Process

1.  We will acknowledge receipt of your report within 48 hours.
2.  We will investigate the issue and provide an estimated timeline for a fix.
3.  We will notify you once the vulnerability has been addressed.

### Responsible Disclosure

We ask that you follow responsible disclosure principles and allow us a reasonable amount of time to fix the issue before sharing any information publicly.
